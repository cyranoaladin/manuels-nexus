# Charte v3.2 — design validé

Le spécimen NSI est le banc de contrôle de la charte. La v3.2 fige les titres d'encadrés en couleur explicite, les accents du contenu et l'ASCII du code par deux gates de corpus. L'ouverture devient une composition identitaire complète (numéro, motif Types construits, titre et signature) ; les pages intérieures récupèrent leur mini-signature et leur onglet.

Les changements restent centralisés : la classe porte les encadrés et l'ouverture, les extensions portent le code et les signatures, et le spécimen couvre les huit encadrés, le contrat, les exercices, le QCM et la figure. Les gates retirent les environnements de code avant de rechercher les accents et ne recherchent l'ASCII interdit qu'à l'intérieur de ces environnements.

La preuve de livraison consiste en tests de régression, `make lot-gates`, compilation du spécimen et du pilote sans overfull, puis export PNG 150 dpi inspecté page par page.
