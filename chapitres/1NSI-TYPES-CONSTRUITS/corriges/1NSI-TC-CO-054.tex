\begin{corrige}{1NSI-TC-EX-054}
\begin{enumerate}
  \item Voici la fonction :
\begin{python}
def copie_profonde_grille(grille):
    """Renvoie une copie profonde d'une grille (liste de listes).

    Precondition: grille est une liste de listes.
    """
    resultat = []
    for ligne in grille:
        resultat.append(list(ligne))
    return resultat
\end{python}
On parcourt chaque ligne de la grille et on en crée une copie avec \lstinline{list(ligne)} avant de l'ajouter au résultat.

  \item Vérification :
\begin{python}
g = [[1, 2], [3, 4]]
c = copie_profonde_grille(g)
c[0][0] = 99
print(g[0][0])  # affiche 1
\end{python}
La modification de \lstinline{c} n'affecte pas \lstinline{g} : la copie est bien profonde.

  \item \lstinline{list(grille)} crée une nouvelle liste externe, mais chaque élément (chaque ligne) reste une \textbf{référence} vers la même sous-liste. C'est une copie superficielle. Pour que la copie soit profonde, il faut aussi copier chaque sous-liste individuellement, ce que fait \lstinline{list(ligne)} dans la boucle.
\end{enumerate}
\end{corrige}
