\subsection{Situation-problème concrète}

Une station météo stocke des coordonnées fixes, des relevés horaires
modifiables et des mesures accessibles par nom.

\subsection{Activité d’entrée}

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  Identifier ce qui doit rester immuable dans un tuple.
\item
  Modifier une liste de températures.
\item
  Lire une clé dans un dictionnaire de station.
\item
  Décrire ce qui se passe avec une liste vide.
\end{enumerate}

\subsection{Consigne technique détaillée}

\begin{itemize}
\tightlist
\item
  Problème à programmer : Construire un résumé de mesures avec tuple de
  localisation, liste de valeurs et dictionnaire de métadonnées.
\item
  Starter code : \texttt{code/P04\_starter\_types\_construits.py}.
\item
  Tests attendus :
  \texttt{code/P04\_tests\_attendus\_types\_construits.py}.
\item
  Corrigé professeur séparé :
  \texttt{code/P04\_corrige\_professeur\_types\_construits.py}.
\item
  Livrable vérifiable : fichier Python complété, sortie de tests
  nominal, limite et invalide, puis commentaire de deux lignes sur le
  cas limite.
\item
  Exemple d'exécution : lancer les tests avec \texttt{TP\_MODULE}
  pointant vers le module à contrôler.
\item
  Cas limite principal : tentative de modification interdite. \#\#
  Étapes de réalisation
\item
  Étape 1 : coder ou tester tuple de coordonnées à partir de
  \texttt{(36.8,\ 10.2)}, puis contrôler tentative de modification
  interdite.
\item
  Étape 2 : coder ou tester liste de relevés à partir de
  \texttt{{[}18,\ 20,\ 19{]}}, puis contrôler liste vide.
\item
  Étape 3 : coder ou tester dictionnaire à partir de
  \texttt{\{"nom":\ "A",\ "temperature":\ 21\}}, puis contrôler clé
  absente.
\item
  Étape 4 : coder ou tester copie de liste à partir de
  \texttt{{[}{[}1{]},\ {[}2{]}{]}}, puis contrôler liste imbriquée. \#\#
  Tests attendus
\item
  Test nominal : donnée ordinaire issue du premier exemple.
\item
  Test limite : entrée minimale, vide ou borne de représentation.
\item
  Test invalide : type ou valeur explicitement refusé par la
  spécification. \#\# Exercices numérotés
\item
  Exercice 1 : résoudre tuple de coordonnées avec \texttt{(36.8,\ 10.2)}
  ; attendu : coordonnées conservées.
\item
  Exercice 2 : expliquer liste de relevés à partir de
  \texttt{{[}18,\ 20,\ 19{]}} ; attendu : \texttt{19}.
\item
  Exercice 3 : comparer dictionnaire avec
  \texttt{\{"nom":\ "A",\ "temperature":\ 21\}} ; attendu : \texttt{21}
  pour \texttt{temperature}.
\item
  Exercice 4 : corriger copie de liste pour
  \texttt{{[}{[}1{]},\ {[}2{]}{]}} ; attendu : modification locale
  contrôlée.
\item
  Exercice 5 : tester un cas limite lié à tentative de modification
  interdite ; attendu : le comportement de tuple de coordonnées est
  contrôlé.
\item
  Exercice 6 : classer deux méthodes possibles pour liste de relevés ;
  attendu : la méthode robuste est choisie et justifiée.
\item
  Exercice 7 : justifier un transfert qui utilise dictionnaire avec une
  donnée nouvelle ; attendu : la justification reste valable sur le
  nouveau cas.
\item
  Exercice 8 : étendre un énoncé volontairement erroné sur copie de

\subsection{Erreurs fréquentes}

\begin{itemize}
\tightlist
\item
  Erreur fréquente EF1 - Modifier un tuple comme une liste.
\item
  Erreur fréquente EF2 - Parcourir les indices quand les valeurs
  suffisent.
\item
  Erreur fréquente EF3 - Accéder à une clé sans vérifier sa présence.
\item
  Erreur fréquente EF4 - Copier une liste imbriquée seulement au premier
  niveau.
\end{itemize}

\subsection{Remédiation ciblée}

\begin{itemize}
\tightlist
\item
  Activité corrective EF1 : Identifier mutabilité et usage avant
  d'écrire une affectation.
\item
  Activité corrective EF2 : Écrire deux boucles, avec indices puis avec
  valeurs, et comparer.
\item
  Activité corrective EF3 : Tester \texttt{cle\ in\ dictionnaire} avant
  la lecture.
\item
  Activité corrective EF4 : Modifier une sous-liste et observer l'effet
  sur la copie.
\end{itemize}

\subsection{Complément TP - fonctions sur types construits}

\subsubsection{Consigne technique détaillée}

Implémenter trois fonctions dans le starter :

\begin{python}
def milieu(a: tuple[float, float], b: tuple[float, float]) -> tuple[float, float]:
    ...

def stations_chaudes(stations: list[dict], seuil: int) -> list[str]:
    ...

def moyenne_notes(notes: list[int]) -> float:
    ...
\end{python}

\subsubsection{Tests attendus}

\begin{itemize}
\tightlist
\item
  \texttt{milieu((0,\ 4),\ (6,\ 10))} renvoie \texttt{(3.0,\ 7.0)}.
\item
  \texttt{stations\_chaudes({[}\{"nom":\ "A",\ "temperature":\ 21\},\ \{"nom":\ "B",\ "temperature":\ 18\}{]},\ 20)}
  renvoie \texttt{{[}"A"{]}}.
\item
  \texttt{moyenne\_notes({[}{]})} lève \texttt{ValueError}.
\item
  \texttt{milieu((2,),\ (4,\ 5))} lève \texttt{ValueError} car le tuple
  n'a pas deux coordonnées.
\end{itemize}

\subsubsection{Livrable vérifiable}

Le fichier rendu contient les fonctions, quatre assertions exécutables
et une phrase qui justifie tuple immuable, liste mutable et dictionnaire
par clé.
