\subsection{Exemples corrigés}

\subsubsection{Exemple corrigé 1 - Exemple principal}

Le tuple \texttt{point\ =\ (3,\ 5)} regroupe deux coordonnées liées. La
fonction \texttt{milieu((1,\ 2),\ (5,\ 8))} renvoie
\texttt{((1+5)/2,\ (2+8)/2)}, donc \texttt{(3.0,\ 5.0)}. On ne modifie
pas directement \texttt{point{[}0{]}} : si une nouvelle coordonnée est
nécessaire, on construit un nouveau tuple. \#\#\# Exemple corrigé 2 -
Contrôle ou contre-exemple La liste
\texttt{temperatures\ =\ {[}18,\ 20,\ 19{]}} accepte
\texttt{temperatures{[}1{]}\ =\ 21}, car elle est mutable. Le tuple
\texttt{coord\ =\ (18,\ 20)} refuserait \texttt{coord{[}1{]}\ =\ 21}. Le
choix de structure dépend donc de l'opération attendue : modifier une
série ou figer un couple de valeurs.

\subsection{Erreurs fréquentes}

\begin{itemize}
\tightlist
\item
  Confondre le vocabulaire de tuples avec une simple récitation :
  corriger par un exemple calculé ou exécuté.
\item
  Oublier une hypothèse de tuples, listes et dictionnaires : corriger en
  l'écrivant avant la méthode.
\item
  Conclure sans contrôle sur types construits : corriger par un cas
  limite ou une vérification inverse.
\end{itemize}

\subsection{Cas limites}

\begin{itemize}
\tightlist
\item
  Cas de départ vide ou nul pour tuples, à traiter selon la convention
  du chapitre P04.
\item
  Donnée invalide dans tuples, listes et dictionnaires, par exemple
  symbole interdit, clé absente ou requête trop large selon la fiche.
\item
  Cas frontière de types construits où une seule valeur change la
  méthode ou le résultat attendu.
\end{itemize}

\subsection{Réponses rapides}

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  La méthode attendue pour tuples commence par les données puis applique
  l'opération du chapitre P04.
\item
  L'erreur vient de l'hypothèse manquante ; elle se corrige en testant
  le cas mentionné dans tuples, listes et dictionnaires.
\item
  Le cas limite doit donner un résultat explicite, par exemple 0, vide,
  absent ou hors plage selon types construits.
\item
  Le contrôle compare le résultat avec la définition ou avec une
  opération inverse de tuples.
\end{enumerate}
